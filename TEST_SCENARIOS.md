# Тестовые сценарии — 55 PT-пресетов + прямые UART тесты

## Как тестировать

### Тест 1: Arduino → Новый Дисплей (прямая интеграция)

1. Открой Serial Monitor Arduino (115200 baud)
2. Введи команду `PT:N` (например `PT:1`) и нажми Enter
3. Arduino устанавливает внутренние переменные и отправляет их на новый дисплей через UART
4. **Проверь**: на новом дисплее должны появиться значения из колонки "Ожидаемые значения"

### Тест 2: Новый Дисплей → Arduino (обратная интеграция)

1. Введи `PT:N` в Arduino Serial Monitor для установки режима
2. На новом дисплее нажимай кнопки режима (M1-M8) и подрежима (S1-S3) через тачскрин
3. **Проверь**: старый LCD дисплей должен обновить режим. Arduino Serial Monitor покажет `DISP RX: <TOUCH:Mx>`

### Тест 3: Прямые UART команды

- Вводи команды прямо в Serial Monitor Arduino для проверки конкретных состояний
- Формат: `<CMD:PARAMS>` через UART ESP32←Arduino
- Для тестирования лимитов, алертов, RPM — использовать секцию "Прямые UART тесты" ниже

## ⚠️ Знаки позиций Z/X

Старый LCD отображает позиции с **инверсией знака**:

- Если `Size_Z_mm > 0` → LCD показывает `Z-XX.XX` (минус!)
- Если `Size_Z_mm <= 0` → LCD показывает `Z+XX.XX` (плюс!)

Новый дисплей: UartDisplay.ino отправляет `-(Size_Z_mm)×10`, поэтому новый дисплей совпадает со старым LCD.
**Знаки в таблице ниже — правильные для нового дисплея (совпадают со старым LCD).**

---

## ⚠️ Формат Проходов/Заходов (новый дисплей)

Новый дисплей показывает N/M формат "выполнено / всего" двумя цветами:

- **Жёлтый** = выполнено (Pass_Nr) — N (текущий)
- **Зелёный** = всего (Pass_Total или Pass_Total_Sphr) — M (итого)

Примеры: `1/6` = 1 выполнен из 6 всего. `--` = нет данных (Pass_Total=0).

Старый LCD показывал: *сколько осталось* = Pass_Total - Pass_Nr + 1 (для Нар/Внутр).

## M3 — РЕЗЬБА (7+6 сценариев)

> Новый дисплей M3:
>
> - **Большое значение** = название резьбы из таблицы (напр. `1.50mm`, `8tpi`, `G  1/8`)

## BUG-04: M3 Edit Mode Sync — резьбы с одинаковым mm_x100

Эти сценарии проверяют BUG-04: при входе в edit mode (тап на большое значение) `local_step`
должен синхронизироваться по `thread_name`, а не только по `mm_x100`.

**Как тестировать**: ESP32 тест-режим (5× DN) → перейти к сценарию 36/37/38 →
тапнуть большое значение (входит edit mode) → нажать UP один раз →
`local_step` должен переместиться на 1 шаг вверх по таблице (а не прыгнуть в другую категорию).


| #   | Тест            | thread_mm | thread_name | Ожидаемый edit-sync | Предыдущий шаг в таблице |
| --- | --------------- | --------- | ----------- | ------------------- | ------------------------ |
| D05 | BUG04/D05 64tpi | 40        | `64tpi`     | idx=44 (64tpi)      | idx=43 (72tpi)           |
| D06 | BUG04/D06 G 1/8 | 91        | `G 1/8`     | idx=48 (G 1/8)      | idx=47 (G 1/16)          |
| E04 | BUG04/E04 K 1/8 | 94        | `K 1/8`     | idx=58 (K 1/8)      | idx=57 (K 1/16)          |

## Прямые UART тесты — Лимиты упоров

Отправляй через Arduino Serial Monitor. Arduino должен получить и переслать на ESP32 при следующем цикле
(или: отправь напрямую в ESP32 UART если есть доступ к `/dev/cu.usbmodem14201`).

**Формат команды от Arduino к ESP32**: `<LIMITS:L,R,F,B>` где 1=установлен, 0=нет.

**Индикаторы на дисплее**: 4 стрелки в строке состояния (x=312,332,352,372):

- `←` (Z Лево): **cyan** `#00d4ff` когда SET, тускло `#1a3a4a` когда нет
- `→` (Z Право): **cyan** `#00d4ff` когда SET, тускло когда нет
- `↑` (X Вперёд): **зелёный** `#00ff88` когда SET, тускло `#1a3a1a` когда нет
- `↓` (X Назад): **зелёный** `#00ff88` когда SET, тускло когда нет

### L-01: Все лимиты выключены

```
<LIMITS:0,0,0,0>
```

Ожидание: все 4 стрелки тусклые.

### L-02: Только Z-Лево

```
<LIMITS:1,0,0,0>
```

Ожидание: ← cyan, остальные тусклые.

### L-03: Только Z-Право

```
<LIMITS:0,1,0,0>
```

Ожидание: → cyan, остальные тусклые.

### L-04: Оба Z-лимита

```
<LIMITS:1,1,0,0>
```

Ожидание: ← и → cyan, ↑ и ↓ тусклые.

### L-05: Только X-Вперёд

```
<LIMITS:0,0,1,0>
```

Ожидание: ↑ зелёный, остальные тусклые.

### L-06: Только X-Назад

```
<LIMITS:0,0,0,1>
```

Ожидание: ↓ зелёный, остальные тусклые.

### L-07: Оба X-лимита

```
<LIMITS:0,0,1,1>
```

Ожидание: ↑ и ↓ зелёные, Z тусклые.

### L-08: Все 4 лимита

```
<LIMITS:1,1,1,1>
```

Ожидание: все 4 стрелки яркие (← → cyan, ↑ ↓ зелёные).

### L-09: Диагональ Z-Лево + X-Назад

```
<LIMITS:1,0,0,1>
```

Ожидание: ← cyan, ↓ зелёный, → и ↑ тусклые.

### L-10: Диагональ Z-Право + X-Вперёд

```

```

## Прямые UART тесты — RPM и мотор

### R-01: Высокий RPM в M3

```
PT:11
<RPM:1500>
```

Ожидание: в M3 row3 (ОБ/МИН) показывает значение Rpm_Limit из Thread_Info[11].

### R-02: Мотор включён

```
<STATE:1,1,1>
```

Ожидание: индикатор мотора активен в строке состояния.

### R-03: Мотор выключен

```
<STATE:0,1,1>
```

Ожидание: индикатор мотора неактивен.

## Последовательности смены режимов

### S-01: Полный обход всех режимов

```
PT:1  → PT:6  → PT:11 → PT:18 → PT:22 → PT:26 → PT:30 → PT:34
```

При каждом переходе:

- Проверить заголовок режима (M1..M8)
- Проверить подпись secondary (ПРОХОДЫ/ЦИКЛОВ/ДЕЛЕНИЙ/УГОЛ°)
- Проверить что позиции Z/X отображаются корректно

### S-02: Смена режима с активным алертом

```
PT:28        ← M6/S1 — алерт "Режим невозможен"
PT:22        ← M5/S1 — алерт должен исчезнуть, нет нового алерта
PT:28        ← M6/S1 — алерт снова появляется
PT:27        ← M6/S3 — алерт исчезает
```

### S-03: THR_CAT полный цикл в M3

```
PT:11        ← M3 метрическая 1.00mm
```

Потом 4× double-tap на подпись типа:

- Double-tap 1: метр → дюйм (должно показать первую дюймовую)
- Double-tap 2: дюйм → G-труб (G 1/16)
- Double-tap 3: G-труб → K-труб (K 1/16)
- Double-tap 4: K-труб → метр (0.20mm)

## Чеклист тестирования обратной интеграции (Дисплей → Arduino)

После `PT:1` (или любого другого пресета):


| Действие на тачскрине     | Ожидаемый результат Arduino | UART команда       |
| ------------------------- | --------------------------- | ------------------ |
| Нажать **M1**             | Старый LCD: режим ПОДАЧА    | `<TOUCH:M1>`       |
| Нажать **M2**             | Старый LCD: режим аПОДАЧА   | `<TOUCH:M2>`       |
| Нажать **M3**             | Старый LCD: режим РЕЗЬБА    | `<TOUCH:M3>`       |
| Нажать **M4**             | Старый LCD: режим КОНУС <   | `<TOUCH:M4>`       |
| Нажать **M5**             | Старый LCD: режим КОНУС >   | `<TOUCH:M5>`       |
| Нажать **M6**             | Старый LCD: режим ШАР       | `<TOUCH:M6>`       |
| Нажать **M7**             | Старый LCD: режим ДЕЛИЛКА   | `<TOUCH:M7>`       |
| Нажать **M8**             | Старый LCD: режим РЕЗЕРВ    | `<TOUCH:M8>`       |
| Нажать **S1**             | Подрежим: Внутренний        | `<TOUCH:S1>`       |
| Нажать **S2**             | Подрежим: Ручной            | `<TOUCH:S2>`       |
| Нажать **S3**             | Подрежим: Наружный          | `<TOUCH:S3>`       |
| Нажать **UP** (парам)     | Увеличить параметр          | `<TOUCH:PARAM_UP>` |
| Нажать **DN** (парам)     | Уменьшить параметр          | `<TOUCH:PARAM_DN>` |
| Нажать **OK**             | Следующий элемент меню      | `<TOUCH:PARAM_OK>` |
| Нажать **OK** на алерте   | Закрыть алерт, Switch_Man() | `<TOUCH:ALERT_OK>` |
| Нажать **THR_CAT** (в M3) | Переключить: метр→дюйм→G→K  | `<TOUCH:THR_CAT>`  |

## Sub-Edit режим (тап на строку правой панели)


| Действие                         | Ожидание                                      |
| -------------------------------- | --------------------------------------------- |
| Тап на СЪЁМ (row3 M1/M2)         | Жёлтая рамка вокруг row3, UART: `SUBSEL:AP`   |
| Тап на КОНУС (row3 M4/M5)        | Жёлтая рамка вокруг row3, UART: `SUBSEL:CONE` |
| Тап на ДЕЛЕНИЙ (row1 M7)         | Жёлтая рамка вокруг row1, UART: `SUBSEL:DIVN` |
| Тап на МЕТКА (row2 M7)           | Жёлтая рамка вокруг row2, UART: `SUBSEL:DIVM` |
| UP/DN при активном sub-edit      | Значение меняется + Arduino получает UART     |
| OK или 5 сек таймаут             | Рамка исчезает, sub-edit выходит              |
| Тап на row3 M3 (ОБ/МИН / ХОД ММ) | Ничего (read-only, нет SUBSEL:PH)             |

## Единицы измерения (Arduino-сторона)


| Переменная        | Тип      | Единица          | Пример                                |
| ----------------- | -------- | ---------------- | ------------------------------------- |
| `Feed_mm`         | uint16_t | 0.01 мм/об       | 10 = 0.10 мм/об                       |
| `aFeed_mm`        | uint16_t | целое            | 10 → новый дисплей: `10` (без дробей) |
| `Thread_Step`     | byte     | индекс таблицы   | 13 = 1.50мм; 22 = 8tpi; 48 = G 1/8    |
| `Size_Z_mm`       | long     | 0.01 мм          | 5678 → новый дисплей: `-56.78`        |
| `Size_X_mm`       | long     | 0.01 мм          | -850 → новый дисплей: `+8.50`         |
| `Ap`              | int      | 0.01 мм          | 15 = 0.15мм                           |
| `Ph`              | int      | кол-во заходов   | 2 = двузаходная                       |
| `Sph_R_mm`        | long     | 0.01 мм (радиус) | 2000 = ø40мм                          |
| `Bar_R_mm`        | long     | 0.01 мм (радиус) | 500 → ножка = 500×2/100 = **10.00мм** |
| `Pass_Total_Sphr` | uint16_t | кол-во заходов   | 80 → заходов на новом: **80**/Nr      |
| `Total_Tooth`     | byte     | кол-во делений   | макс 255                              |
| `Spindle_Angle`   | ulong    | 0.1°             | 600 = 60.0°                           |
| `Cone_Step`       | byte     | индекс таблицы   | 0=45°, 45=KM0, 51=KM6, 57=1:20        |


> **Инверсия знаков Z/X**: Print.ino отображает позиции с инвертированным знаком.
> UartDisplay.ino компенсирует это, отправляя `-(Size_Z_mm) × 10`.
