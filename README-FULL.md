## 1. Параметры токарного станка

Ключевые параметры задаются в начале файла `7e2_Mod_LCD4.3_v1.3.ino`:

```cpp
// Энкодер шпинделя
#define ENC_LINE_PER_REV     1800   // Линий энкодера на 1 оборот шпинделя

// Ось Z (продольная)
#define MOTOR_Z_STEP_PER_REV 200    // Шагов двигателя на оборот винта Z (1.8° ШД)
#define SCREW_Z              200    // Шаг продольного винта Z, сотки (200 = 2.00мм)
#define McSTEP_Z             4      // Микрошаг Z (4 = 1/4, итого 800 шагов/об)

// Ось X (поперечная)
#define MOTOR_X_STEP_PER_REV 200    // Шагов двигателя на оборот винта X
#define SCREW_X              100    // Шаг поперечного винта X, сотки (100 = 1.00мм)
#define McSTEP_X             4      // Микрошаг X (4 = 1/4, итого 800 шагов/об)

// Зазоры и допуски
#define REBOUND_X            200    // Отскок X в микрошагах (должен быть больше люфта)
#define REBOUND_Z            200    // Отскок Z в микрошагах

// Ограничения подачи
#define MIN_FEED             1      // Минимальная подача, сотки/об (0.01мм/об)
#define MAX_FEED             25     // Максимальная подача, сотки/об (0.25мм/об)
#define MIN_aFEED            10     // Мин. асинхронная подача, мм/мин
#define MAX_aFEED            400    // Макс. асинхронная подача, мм/мин

// Нарезание резьбы
#define PASS_FINISH          2      // Количество чистовых проходов после цикла
#define THRD_ACCEL           24     // Коэффициент ускорения на резьбах
```

### Переключатель режима (M1–M7)

| Pin Mega | Регистр | Режим |
|----------|---------|-------|

| Pin Mega | Назначение |
|----------|-----------|
| Pin 16 (PH1) | Buzzer (активный, ACTIVE-LOW) |

| Pin Mega | Назначение |
|----------|-----------|
| **Pin 18 (TX1)** | Данные → дисплей (JC4827W543 RX) |
| **Pin 19 (RX1)** | Данные ← дисплей (JC4827W543 TX) |
| GND | Общая земля |

> Serial1, скорость **115200 baud**

## 3. Подключение нового дисплея JC4827W543

Дисплей JC4827W543 (4.3" 480×272, ESP32-S3 + NV3041A QSPI + GT911 Touch) подключается к Arduino Mega через UART:

```
Arduino Mega                JC4827W543
─────────────────           ──────────────────
Pin 18 (TX1) ──────────────► RX
Pin 19 (RX1) ◄────────────── TX
GND ──────────────────────── GND
```

> Питание дисплея — от отдельного источника 5В/1А (USB или DC-DC).
> Уровни логики: Arduino 5В → дисплей 3.3В. Если нет встроенного делителя — используй резисторный делитель или level-shifter на линии TX Arduino → RX дисплея.

### Пины ESP32-S3 внутри JC4827W543 (справочно)

Эти пины уже разведены внутри модуля, конфигурируются через `platformio.ini`:

| Группа | Пин ESP32-S3 | Назначение |
|--------|-------------|-----------|
| Дисплей QSPI | 45, 47, 21, 48, 40, 39 | CS, SCK, D0–D3 |
| Подсветка | 1 | BL (PWM) |
| Touch I2C SCL | 4 | GT911 SCL |
| Touch I2C SDA | 8 | GT911 SDA |
| Touch RST | 38 | GT911 Reset |
| Touch INT | 3 | GT911 Interrupt |
| **UART RX** | (назначается в коде) | От Arduino Mega TX1 |
| **UART TX** | (назначается в коде) | К Arduino Mega RX1 |

## 4. Сборка прошивки Arduino

### Требования

- Arduino IDE 2.x или arduino-cli
- Плата: **Arduino Mega 2560** (arduino:avr:mega)
- Зависимости: нет внешних библиотек (всё включено в проект)

### Через Arduino IDE

1. Открой файл `7e2_Mod_LCD4.3_v1.3.ino`
2. Выбери плату: **Tools → Board → Arduino Mega 2560**
3. Выбери порт (обычно `/dev/cu.usbserial-XXXX` на macOS, `COMx` на Windows)
4. Нажми **Upload** (Ctrl+U)

### Через arduino-cli

```bash
# Компиляция
arduino-cli compile --fqbn arduino:avr:mega /path/to/7e2_Mod_LCD4.3_v1.3

# Загрузка
arduino-cli upload --fqbn arduino:avr:mega --port /dev/cu.usbserial-1480 /path/to/7e2_Mod_LCD4.3_v1.3
```

> macOS путь к arduino-cli (если установлен через Arduino IDE 2.x):
> `/Applications/Arduino IDE.app/Contents/Resources/app/lib/backend/resources/arduino-cli`

## 5. Сборка прошивки ESP32 (дисплей)

### Требования

- [PlatformIO](https://platformio.org/) (CLI или VSCode расширение)
- Python 3.x с pip

### Команды

```bash
cd ELS_Display

# Только компиляция (проверка)
~/.platformio/penv/bin/pio run -e jc4827w543

# Компиляция + загрузка по USB
~/.platformio/penv/bin/pio run -e jc4827w543 --target upload

# Мониторинг Serial (USB CDC)
~/.platformio/penv/bin/pio device monitor -e jc4827w543
```

> ESP32-S3 использует USB CDC — дисплей появится как `/dev/cu.usbmodem14201` (macOS)
> или `COMx` (Windows, нужен драйвер CDC).

### Шрифты (если нужна перегенерация)

LVGL шрифты в `ELS_Display/src/fonts_bold/` — Tahoma Bold, различные размеры.

```bash
# Инструмент генерации шрифтов (lv_font_conv)
node ~/.nvm/versions/node/v20.20.0/lib/node_modules/lv_font_conv/lv_font_conv.js \
  --no-compress --bpp 4 --size 72 \
  --font "/System/Library/Fonts/Supplemental/Tahoma Bold.ttf" \
  --range "0x002B,0x002D-0x003A" \
  --format lvgl -o src/fonts_bold/font_tahoma_bold_72.c
```

> Используй системный `/System/Library/Fonts/Supplemental/Tahoma Bold.ttf`.
> Wine-версия `tahomabd.ttf` даёт другие глифы — не использовать!

## 6. UART протокол Arduino ↔ ESP32

### Формат пакета

```
<CMD:VALUE>\n
```

Примеры:
```
<MODE:1>          — режим M1
<FEED:15>         — подача 0.15мм/об (x100)
<POS_Z:+56780>    — позиция Z +56.780мм (x1000)
<THREAD_NAME:1.50mm>
<LIMITS:1,1,0,0>  — левый и правый лимиты установлены
```

### Команды Arduino → ESP32 (основные)

| Команда | Формат | Описание |
|---------|--------|---------|
| `MODE` | `<MODE:N>` | Режим 1–7 |
| `SUBMODE` | `<SUBMODE:N>` | Подрежим 1=Int, 2=Man, 3=Ext |
| `SELFMENU` | `<SELFMENU:N>` | SM1/SM2/SM3 (SelectMenu) |
| `FEED` | `<FEED:N>` | Подача (мм*100) |
| `AFEED` | `<AFEED:N>` | Асинхронная подача (мм/мин) |
| `POS_Z` | `<POS_Z:±N>` | Позиция Z (мкм, 1/1000 мм) |
| `POS_X` | `<POS_X:±N>` | Позиция X (мкм, 1/1000 мм) |
| `RPM` | `<RPM:N>` | Обороты шпинделя |
| `AP` | `<AP:N>` | Съём (мм*100) |
| `PASS` | `<PASS:N/M>` | Проход N из M |
| `LIMITS` | `<LIMITS:L,R,F,B>` | Концевики (0/1) |
| `THREAD_NAME` | `<THREAD_NAME:1.50mm>` | Имя резьбы (8 символов) |
| `THREAD_CYCLES` | `<THREAD_CYCLES:N>` | Осталось циклов резьбы |
| `RPM_LIMIT` | `<RPM_LIMIT:N>` | Лимит RPM для резьбы |
| `CONE` | `<CONE:N>` | Индекс конуса |
| `DIVN` | `<DIVN:N>` | Делитель: делений |
| `DIVM` | `<DIVM:N>` | Делитель: текущая метка |
| `SPHERE` | `<SPHERE:N>` | Радиус шара (мм*100) |
| `ALERT` | `<ALERT:N>` | Всплывающее сообщение 1–4 |

### Команды ESP32 → Arduino (от тачскрина)

| Команда | Описание |
|---------|---------|
| `<TOUCH:M1>` .. `<TOUCH:M7>` | Переключение режима |
| `<TOUCH:S1>` .. `<TOUCH:S3>` | Переключение подрежима |
| `<TOUCH:KEY:UP>` / `<TOUCH:KEY:DN>` | UP/DOWN (навигация, параметры) |
| `<TOUCH:PARAM_OK>` | SELECT (SM1→SM2→SM3) |
| `<TOUCH:THR_CAT>` | Переключение категории резьбы |
| `<TOUCH:AP:N>` | Установить съём N (мм*100) |
| `<TOUCH:CONE:N>` | Установить индекс конуса |
| `<TOUCH:DIVN:N>` | Установить кол-во делений |
| `<TOUCH:DIVM:N>` | Установить текущую метку |
| `<TOUCH:SPHERE:N>` | Установить радиус шара |
| `<TOUCH:THREAD_STEP:N>` | Выбрать резьбу по индексу |
| `<TOUCH:JOY:LEFT/RIGHT/UP/DOWN>` | Джойстик: перемещение |
| `<TOUCH:JOY:STOP>` | Джойстик: остановить |
| `<TOUCH:JOY:RAPID_ON/OFF>` | Быстрое перемещение |
| `<TOUCH:EMUL:STOP_L/R/F/B>` | Эмуляция концевика |
| `<TOUCH:ALERT_OK>` | Подтверждение алерта |
| `<READY>` | ESP32 готов (запрос состояния) |
| `<PONG>` | Ответ на PING (запрос состояния) |

## 7. Режимы работы и управление с тачскрина

### Режимы (M1–M7)

| Режим | Кнопка | Название | Основной параметр |
|-------|--------|---------|-------------------|
| M1 | `M1` | Синхронная подача | Подача (мм/об) |
| M2 | `M2` | Асинхронная подача | Подача (мм/мин) |
| M3 | `M3` | Резьба | Шаг резьбы |
| M4 | `M4` | Конус левый | Угол конуса |
| M5 | `M5` | Конус правый | Угол конуса |
| M6 | `M6` | Шар | Радиус шара |
| M7 | `M7` | Делитель | Делений/Метка |

### Подрежимы (S1–S3)

| Подрежим | Переключатель | Описание |
|---------|--------------|---------|
| S1 Внутренний | `S1` | Авто-цикл внутренней обработки |
| S2 Ручной | `S2` | Ручное управление, синхронизация |
| S3 Наружный | `S3` | Авто-цикл наружной обработки |

### SelectMenu SM1/SM2/SM3

Переход SM1→SM2→SM3 через:
- **Физическая кнопка SELECT** (Pin A4)
- **Двойной тап** по ячейке ПРОХОДЫ/ЦИКЛОВ на тачскрине (row 6)
- **Двойной тап** по ячейке СЪЁМ/ТИП/ШАР ММ (нижний ряд правой панели)

| SM | M1/M2/M4/M5 | M3 Резьба | M6 Шар |
|----|-------------|-----------|--------|
| SM1 | Параметры: Ap (UP/DN) | Выбор резьбы (UP/DN) | Радиус (UP/DN) |
| SM2 | Сброс X/Z позиции | Заходы резьбы Ph | Ширина резца |
| SM3 | Натяжение/отскок (Ext) | Сброс позиции | Сброс позиции |

### Навигация на тачскрине

| Действие | Результат |
|---------|----------|
| Тап по ячейке ПОЗИЦИЯ Z/X | Войти в sub-edit (редактирование параметра) |
| Тап по ячейке СЪЁМ/КОНУС/ШАР ММ | Войти в sub-edit для параметра |
| Двойной тап ПОЗИЦИЯ Z (M3) | Переключить категорию резьбы (Метр/Дюйм/G/K) |
| Двойной тап ПОЗИЦИЯ X | Показать/скрыть джойстик-оверлей |
| Двойной тап по значению резьбы (M3) | Показать меню категорий резьбы |
| Двойной тап ПРОХОДЫ/ЦИКЛОВ | SM1→SM2→SM3 |
| UP/DOWN кнопки | Навигация и изменение параметра в sub-edit |
| OK кнопка | Выйти из sub-edit / подтвердить (M7: сброс позиции) |

### Таблица резьб M3

67 резьб в 4 категориях:

| Категория | Индексы | Примеры |
|-----------|---------|---------|
| Метрические | 0–19 | 0.50mm, 0.75mm, 1.00mm ... 4.00mm |
| Дюймовые | 20–46 | 8tpi, 10tpi, 11tpi ... 80tpi |
| G-трубные | 47–56 | G 1/8, G 1/4, G 3/8 ... G 2" |
| K-трубные | 57–66 | K 1/8, K 1/4 ... K 2" |

### Индикаторы концевых выключателей (статусбар)

```
← → ↑ ↓
```

| Символ | Ось | Состояние |
|--------|-----|----------|
| ← (cyan) | Z-Left | Лимит установлен |
| → (cyan) | Z-Right | Лимит установлен |
| ↑ (green) | X-Front | Лимит установлен |
| ↓ (green) | X-Rear | Лимит установлен |
| dim | любой | Лимит не установлен |

## 8. Отладка через Serial Monitor

Arduino Mega Serial Monitor (115200 baud) принимает текстовые команды для эмуляции физических входов.

### Команды Serial Monitor

```
=== РЕЖИМ И ПОДРЕЖИМ ===
M1 .. M7       — переключить режим
S1 / S2 / S3  — переключить подрежим (Внутр / Ручной / Нар)

=== КНОПКИ МЕНЮ ===
BU             — Button UP
BD             — Button DOWN
BL             — Button LEFT
BR             — Button RIGHT
BS             — Button SELECT (SM1→SM2→SM3)
RAP / RAN      — RAPID нажата / отпущена

=== КОНЦЕВИКИ ===
LL / LR        — Лимит левый / правый (Z-ось)
LF / LB        — Лимит передний / задний (X-ось)

=== РГИ ===
HZ / HX / HN  — Ось РГИ: Z / X / нет
SC1 / SC10     — Масштаб РГИ: x1 / x10

=== ДИСПЛЕЙ ===
SEND           — Отправить всё состояние на ESP32
DPING          — PING к дисплею
AL1..AL4       — Алерт 1–4 на дисплей

=== ТЕСТОВЫЕ ПРЕСЕТЫ ===
PT:N           — Пресет N (1..55), см. TEST_SCENARIOS.md

=== ОТЛАДКА ===
INFO           — Текущее состояние всех переменных
HELP           — Справка по командам
```

### Тестовые пресеты PT:N

Быстрая установка конкретного состояния для тестирования дисплея:

```
PT:1   — M1/S1 Внутр, Подача 0.10, Z=+56.78, X=-8.50, Пр=1/6
PT:5   — M1/S3 Нар с лимитами
PT:10  — M2/S1 Внутр, аПодача 0.10, Пр=2/8
PT:15  — M3/S1 Резьба 1.50mm, Пр=3/12
PT:20  — M3/S2 Резьба G-труба G 1/8
PT:25  — M4/S1 Конус 30°
PT:30  — M5/S2 Конус Морзе 1
PT:35  — M6/S1 Шар R=25.00мм
PT:40  — M7 Делитель 6 делений
...
```

> Все 55 пресетов с точными значениями — в файле [TEST_SCENARIOS.md](TEST_SCENARIOS.md)

### Снятие скриншота с дисплея

```bash
# Отправить 'S' на ESP32 по USB CDC → получить JPEG
python3 ELS_Display/tools/screenshot_usb.py /dev/cu.usbmodem14201
# Скриншот сохраняется в ELS_Display/tools/screenshot_latest.jpg
```

Протокол: `'S'` → магик `0xDEADBEEF` (4 байта) + размер (4 байта LE) + JPEG данные

## 10. Настройка параметров станка

### Изменение шага энкодера

Если у вас энкодер на 600, 1000 или 2500 линий — поменяйте `ENC_LINE_PER_REV`:

```cpp
#define ENC_LINE_PER_REV  600    // для 600-линейного энкодера
```

### Изменение шага ходового винта

```cpp
#define SCREW_Z  200   // 2.00мм/об → изменить на 150 для 1.50мм/об
#define SCREW_X  100   // 1.00мм/об
```

### Изменение микрошага

```cpp
#define McSTEP_Z  8   // 1/8 микрошага → 1600 шагов/об
```

> После изменения `McSTEP_Z/X` — пересчитай `REBOUND_X/Z` (в микрошагах, не мм).

### Количество чистовых проходов для резьбы

```cpp
#define PASS_FINISH  2   // 2 чистовых прохода в конце цикла (можно 1..5)
```
