# Подключение HS800-2 к Arduino Mega (7e2_Mod)

## Схема подключения

```
HS800-2 плата монитора          Arduino Mega 2560
─────────────────────────       ──────────────────
STM32 TX (R78/R79/R80) ──────── pin 15  (RX3)
GND ─────────────────────────── GND
```

**Всего 2 провода.** Никаких делителей напряжения не нужно:
3.3В сигнал HS800-2 читается на 5В входе Mega без проблем.

> **Важно:** pin 14 (TX3) оставить свободным — данные мы только читаем.

### Уже занятые пины (не трогать):
| Пины | Функция |
|------|---------|
| 0/1 | Serial — Serial Monitor / отладка |
| 51/52 | SoftwareSerial → ESP32-S3 JC4827W543 |
| 18/19 | Ручной энкодер |
| 20/21 | I2C (старый LCD, энкодер шпинделя) |

---

## Активация DRO режима

В файле [7e2_Mod_LCD4.3_v1.3.ino](7e2_Mod_LCD4.3_v1.3.ino), строки 4-12,
раскомментируй:
```c
#define USE_DRO_HS800
```

**Без этого define** — прошивка работает в точности как оригинал (шаговые двигатели).

---

## Маппинг осей

| HS800-2 | Гитара 7e2 | Физически |
|---------|-----------|-----------|
| Ось X | Size_X_mm | Суппорт поперечный |
| Ось Y | Size_Z_mm | Каретка продольная |

> На этом станке нет оси Z — поэтому Y линейки HS800-2 соответствует
> оси Z в гитаре (переменная `Size_Z_mm`).

### Если направление перевёрнуто

В [DRO_HS800.ino](DRO_HS800.ino), строки 45-46:
```c
#define DRO_X_SIGN    1   // Изменить на -1 если X движется в обратную сторону
#define DRO_Y_SIGN    1   // Изменить на -1 если Y движется в обратную сторону
```

---

## Единицы

| Источник | Единица | Переменная |
|----------|---------|-----------|
| HS800-2 raw | 0.001мм (мкм) | `dro_pos_x`, `dro_pos_y` |
| 7e2 внутренний | 0.01мм | `Size_X_mm`, `Size_Z_mm` |
| Дисплей JC4827W543 | 0.1мм (×10) | `POS_X`, `POS_Z` команды |

Конвертация: `Size_X_mm = dro_pos_x / 10`

---

## Процедура первого включения

1. Подключи провода согласно схеме выше
2. Раскомментируй `#define USE_DRO_HS800`
3. Залей прошивку на Arduino Mega
4. Включи HS800-2
5. Открой Serial Monitor (115200 бод) — должно появиться:
   ```
   DRO HS800-2: Serial3 @ 57600 (pin 15 RX3)
   ```
6. Подвигай линейку X — на дисплее JC4827W543 должно меняться поле **POS_X**
7. Подвигай линейку Y — должно меняться поле **POS_Z**

## Тест-кейсы для проверки

### TC-1: Режим без DRO (оригинальный)
- `#define USE_DRO_HS800` закомментирован
- HS800-2 можно не подключать
- Ожидание: гитара работает как прежде, позиции X/Z от шаговых двигателей

### TC-2: DRO подключён, позиция X
1. Включить `USE_DRO_HS800`, залить прошивку
2. На HS800-2 ввести X = 25.000 (кнопка X → 2 5 . 0 0 0 → Enter)
3. Ожидание: на JC4827W543 поле POS_X показывает 25.0 (или -25.0 в зависимости от знака)
4. Ожидание: в Serial Monitor — `DISP TX: <POS_X:...>`

### TC-3: DRO подключён, позиция Y
1. На HS800-2 ввести Y = 10.500 (кнопка Y → 1 0 . 5 0 0 → Enter)
2. Ожидание: поле POS_Z на дисплее = 10.5мм

### TC-4: Сброс X в ноль
1. Нажать кнопку ZERO_X на HS800-2
2. Ожидание: поле POS_X на JC4827W543 = 0.000

### TC-5: Сброс Y в ноль
1. Нажать ZERO_Y
2. Ожидание: POS_Z = 0.000

### TC-6: Движение линейки в реальном времени
1. Физически подвинуть линейку X на 10мм
2. Ожидание: POS_X меняется плавно в реальном времени (обновление каждые 250мс)

### TC-7: Потеря связи с HS800-2
1. Отключить провод от HS800-2 пока гитара работает
2. Ожидание: позиция замирает на последнем значении
3. Через 10 секунд в PING: `DRO_OK=0` в Serial Monitor
4. Переподключить — позиция возобновляется автоматически

### TC-8: Существующий функционал не сломан
1. Включить `USE_DRO_HS800`, залить прошивку
2. Переключить режимы: Feed, Thread, Cone, Sphere
3. Ожидание: все режимы работают как прежде
4. Синхронная подача по шпинделю — работает нормально
5. Ручной энкодер — работает нормально

## Известные ограничения

- **Управление двигателями** при DRO режиме: гитара управляет двигателями
  на основе `X_pos` / `Z_pos` (шаговые счётчики), не на основе `Size_X_mm`.
  Значит **функции автоподачи и нарезки резьбы** по-прежнему используют
  шаговые двигатели для управления. DRO только **отображает** позицию.
  Это корректное поведение — DRO является индикатором, не системой управления.

- **OTSKOK и TENSION** (отскок, натяг) всегда считаются от шаговых двигателей
  — это механическая компенсация люфта, не зависящая от DRO.

- **MSize_X_mm** (диаметр): в DRO режиме = `2 × |X|`. Если нужен другой
  расчёт — отредактируй строку в `loop()` в [7e2_Mod_LCD4.3_v1.3.ino](7e2_Mod_LCD4.3_v1.3.ino).
