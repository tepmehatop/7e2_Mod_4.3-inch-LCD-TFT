### Шаг 2: Подключить LCD 2004 через I2C модуль

#### Схема подключения:
```
I2C модуль LCD → Arduino Mega 2560
─────────────────────────────────
VCC  (5V)     →  5V
GND  (GND)    →  GND
SDA           →  Pin 20 (SDA)
SCL           →  Pin 21 (SCL)
```

**ВАЖНО:**
- Pin 20 и Pin 21 на Mega 2560 - это I2C шина (SDA/SCL)
- **ОТКЛЮЧИТЕ старые провода** от пинов 8, 9, 10, 11, 12, 13!
- На I2C модуле есть потенциометр контрастности (синий регулятор)

---

### Шаг 3: Узнать I2C адрес (ВАЖНО!)

Большинство LCD имеют адрес **0x27** или **0x3F**.
В коде установлен **0x27** по умолчанию.

#### I2C Scanner (опционально):
Если не уверены в адресе, запустите сканер:

```cpp
#include <Wire.h>

void setup() {
  Wire.begin();
  Serial.begin(115200);
  Serial.println("\n=== I2C Scanner ===");
}

void loop() {
  byte error, address;
  int nDevices = 0;

  Serial.println("Сканирование...");

  for(address = 1; address < 127; address++) {
    Wire.beginTransmission(address);
    error = Wire.endTransmission();

    if (error == 0) {
      Serial.print("Устройство найдено: 0x");
      if (address < 16) Serial.print("0");
      Serial.println(address, HEX);
      nDevices++;
    }
  }

  if (nDevices == 0)
    Serial.println("Устройства не найдены");
  else
    Serial.println("Сканирование завершено");

  delay(5000);
}
```

Если адрес **0x3F**, измените в строке 254:
```cpp
LCDI2C_Russian lcd(0x3F, 20, 4);  // вместо 0x27
```

#### ❌ Если экран ПОЛНОСТЬЮ ПУСТОЙ:

**Причина 1: Неправильный I2C адрес**
- Запустите I2C Scanner (см. Шаг 3)
- Измените адрес в коде на найденный (0x27 или 0x3F)

**Причина 2: Неправильное подключение**
- Проверьте: SDA → Pin 20, SCL → Pin 21
- Проверьте: VCC → 5V, GND → GND

**Причина 3: Контрастность**
- Покрутите синий потенциометр на I2C модуле

**Причина 4: Задержка инициализации**
- Попробуйте увеличить задержку до 1000мс:
```cpp
_delay_ms(1000);  // вместо 500
lcd.init();
```

#### ❌ Если квадратики ████:

**Причина: Дисплей инициализирован, но неправильный адрес**
- Попробуйте адрес 0x3F вместо 0x27
- Добавьте задержку после init():
```cpp
lcd.init();
_delay_ms(100);
lcd.backlight();
```

#### ❌ Если русские буквы НЕ ПРАВИЛЬНО (�� вместо букв):

**Такого НЕ ДОЛЖНО быть с LCDI2C_Russian!**
Эта библиотека специально для русского языка.

Проверьте:
- Используется класс **LCDI2C_Russian** (не LCDI2C_Generic!)
- Include правильный: `#include <LCDI2C_Multilingual.h>`
